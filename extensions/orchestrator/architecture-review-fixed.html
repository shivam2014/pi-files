Problem. No module owns the scope seam; each surface holds part of the contract, so deleting any piece breaks an invariant.

Solution. Let ScopeManager own extraction, cache, disk I/O, and typed API; scope-guard becomes a thin enforcement adapter with zero domain coupling.

  • ● locality: scope logic in one module
  • ● seam: single adapter boundary
  • ● leverage: enforce at every call site
  • ● depth: interface hides I/O detail
  • ● tests: one surface to verify
  • ● deletion: remove guard without re-implementing scope
ADR. AGENTS.md requires scope-guard.ts to stay zero-coupled to the orchestrator module; keeping it as a file-system adapter preserves this seam.

Problem. Two layers keep their own step/substep models; plan-panel parses rendered lines to convert between them.

Solution. Introduce one event-sourced PlanFeed aggregate; plan-panel and activity-feed become pure renderers of the same model.

  • ● locality: model lives once
  • ● leverage: one aggregate drives both views
  • ● seam: renderer adapter boundary
  • ● depth: feed logic deepens
  • ● tests: no line-parsing stubs
  • ● deletion: remove view without re-implementing conversion
ADR. Layer 1 / Layer 2 separation is a design spec; unifying the model still allows separate render layers.

Problem. registerDelegateTool bundles scope normalization, plan transitions, subagent execution, ask-orchestrator resolution, findings/audit/metrics extraction, and output formatting behind one registration function.

Solution. Split into DelegateController, ScopeExtractor, OutputFormatter, and AskResolver; delegate-tool.ts becomes wiring.

  • ● leverage: formatters reusable
  • ● locality: output logic isolates
  • ● tests: adapters substitutable

Problem. The entry point encodes system prompt authoring, bash-to-tool replacement, subagent tool_call guarding, and extension registration in one default export.

Solution. Extract PromptBuilder, BashInterceptor, and RegistrationHub; index.ts only wires them.

  • ● seam: registration vs policy
  • ● depth: each hub deepens
  • ● leverage: BashInterceptor tested alone
  • ● locality: prompt logic isolates
  • ● tests: swap adapters

Problem. One session.subscribe callback mutates feed state, plan panel detail, peek overlay, timeline, and onUpdate, acting as a hidden router.

Solution. Define a SubagentEvent stream and a small dispatcher; each UI module registers its own handler.

  • ● seam: event stream is boundary
  • ● locality: UI modules own updates
  • ● leverage: add views without touching runner
  • ● depth: dispatcher deepens
  • ● tests: stub stream, not whole state machine

F. plan-panel.ts relies on module-level mutable state

Worth exploring in-process

plan-panel.ts

Before — module scope

After — context-bound

PlanPanel
state per instance

Problem. planState, _setWidget, _sessionId, and timers live in module scope; timers use a globalThis registry to survive hot reload, so concurrent sessions and test isolation are fragile.

Solution. Make PlanPanel a class or context-bound object with explicit lifecycle methods.

  • ● locality: state inside instance
  • ● seam: lifecycle interface
  • ● depth: panel logic deepens
  • ● leverage: one context per session
  • ● tests: injectable
  • ● isolation: no globalThis

Top recommendation

A. Scope lifecycle split across four surfaces

Scope invariants cross the most seams; a single deep module gives the highest leverage for enforcement, deletion safety, and the zero-coupling ADR.
